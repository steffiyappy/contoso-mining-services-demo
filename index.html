Exercise 1 · Daily BriefingResearcher · Critique

How a GM starts the day in 2 minutes. Open Microsoft 365 Copilot at copilot.microsoft.com → click Researcher. Keep the model picker on Auto (this is Critique — dual-model with self-review). Paste the prompt below.

Researcher reads your inbox, calendar, Teams chats, and recent files — then hands General Manager · Operations a single, structured briefing for the day. No more spending the first 30 minutes catching up.

Task 1.1🔍 Researcher · Auto (Critique)

Generate today's GM daily briefing

🅰️ Option A · Wordy briefing (structured text)

Create a daily briefing from emails, messages, documents, and meetings to help me prepare for the current workday. Use your understanding of the projects I work on, workstreams I care about, and key people I work with to understand what is relevant and important. Sections: A short intro paragraph that provides an overview of my day, i.e. what a personal assistant would tell me to greet me in the morning. Key updates since yesterday – Include informational updates from the last workday that are likely to impact my work and collaborations. Suggested tasks for today – Tasks or questions assigned to me that I should handle today. Cross-check that suggested tasks haven't been completed already. List unread email threads from the past week that likely need my reply—prioritize those where I'm on To/CC, @mentioned, or involve requests, deadlines, leadership, or customers. Exclude automated messages. Show only urgent or important items in a concise table, and draft replies where needed. If none require action, say: 'I didn't find any unread threads that require a reply.' Meeting prep for today – Upcoming meetings only, shown in a table with: Meeting (time, title, organizer), Topic Overview, and Suggested Preparations. Note if inferred or not needed. Rules: Start with a header: Daily briefing – Day, Date. Follow with a sub-header stating the timestamp of generation. Start each section with a meaningful emoji. Use only explicitly stated facts; no speculation, interpretations or predictions. If a section has no data, state that. Visual: At the very top of the briefing — above the header — include a single hero illustration that matches the mood of the day's briefing. Pick one of: a sunrise over a mine site, a calm desk-with-coffee scene, or an abstract brand-coloured banner using deep orange (#b45309) and dark slate (#0f172a). Wide aspect ratio (roughly 3:1), no text inside the image. Caption it in small grey text: "Generated illustration · Contoso Mining Services". Do not ask me any clarifying questions. Proceed directly and produce the full briefing in one pass. Keep it tight — no filler.
Reality check on the illustration: Researcher in Microsoft 365 Copilot is primarily a text + tables + citations surface. Inline image generation is best-effort here — on some tenants it renders a real illustration, on others it falls back to a stylised emoji banner or a placeholder line. If the image doesn't appear, follow up with: "Generate the hero illustration described above as a separate image I can download." — that hands it off to the image-gen path.
What success looks like: within 90 seconds you have a one-screen brief: overview paragraph, key updates since yesterday, today's tasks, unread threads that need a reply (with drafts), and a meeting-prep table — all grounded on real signals from your tenant, with citations.

🅱️ Option B · Visual briefing (infographic dashboard)

Same signals, but rendered as a one-screen visual dashboard the GM can glance at over coffee. Run this in Microsoft 365 Copilot Chat (not Researcher — Chat returns inline HTML you can preview).

Generate my GM daily briefing for today as a single-page interactive HTML dashboard, grounded on my emails, Teams chats, calendar and recent files. Layout (all on one screen, no scrolling on a laptop): • Header band: "Daily Briefing — [Day, Date]" with a timestamp on the right. Use brand colours #b45309 (deep orange) and #0f172a (dark slate). • Row 1 — 4 KPI cards: (1) Unread emails needing reply, (2) Meetings today, (3) Open decisions awaiting me, (4) Overdue tasks. Each card: big number, one-line context, RAG colour band (green/amber/red). • Row 2 — two columns: – Left: "Top 5 priorities today" as a vertical list of coloured cards (priority badge: 🔴 high / 🟡 medium / 🟢 low), each with a one-line "what to do". – Right: "Meetings today" timeline — each meeting as a card with time, title, organiser, and a 1-line prep note. • Row 3 — "Unread threads that need a reply": a compact 3-column table (From · Subject · Suggested 1-line reply). Max 5 rows. • Footer: "Generated by Microsoft 365 Copilot · [timestamp]" in small text. Style rules: • All CSS inline, no external fonts, no JavaScript. • Use cards with soft shadow, rounded corners (12px), generous white space. • Use emoji as section icons (📨 📅 ✅ ⚠️). • Use only explicitly stated facts from my tenant. No speculation. If a section has no data, show the card with "Nothing for today" instead of hiding it. • Save the file to OneDrive as DailyBriefing.html. Do not ask me any clarifying questions. Proceed directly and produce the dashboard in one pass.
Reality check: the HTML renders beautifully on screen and as a saved file. Pasting the HTML into Outlook will land as an attachment, not an inline image — if you want to email it, screenshot the rendered page and paste the image, or attach DailyBriefing.html.
Bonus🔍 Researcher · follow-up

Drill in on the single most important item

Pick one item from the briefing that needs deeper thinking. Ask Researcher:

Take the highest-priority item from the briefing above. Pull together everything I've received on it in the last 14 days — emails, chats, files, meeting notes. Summarise the current state in 5 bullets, list the open decisions, and draft one short message I should send today to move it forward.

Exercise 3 · Market Intelligence — Critique & Model CouncilResearcher

When to do this: Right after the daily briefing, when the GM needs deep market intelligence. Open Microsoft 365 Copilot → Researcher.

3.1 · Critique (Auto) — the default for sector intelligence

Researcher on Auto runs Critique mode — a dual-model setup where one model drafts, a second self-reviews the sources, and only the verified report is returned.

Task 3.1🔍 Researcher · Auto (Critique)

Scan the Mining Services & EPC landscape in Indonesia

Research the Mining Services & EPC sector in Indonesia. Benchmark these companies: Petrosea, United Tractors / Pamapersada, Bukit Makmur (BUMA), Saptaindra Sejati, Thiess Indonesia, Macmahon. For each one, compare market share, FY25 revenue growth, EBITDA margin, strategic focus, digital maturity, and recent material moves (M&A, capex, new contracts, regulation). Include 2024–2028 market sizing with the three main growth drivers. Cite every source. Flag any section where the evidence is thin.
Notice the citations and the "reviewed by" stamp — Critique enforces source grounding before the report is shown.

3.2 · Model Council — when the decision is too important for one model

Switch the Researcher model picker to Model Council. Multiple frontier models reason on the same question in parallel — Researcher then reconciles them and shows where they agree, where they diverge, and which sources each one anchored on.

Task 3.2🧑‍⚖️ Researcher · Model Council

Stress-test the strategic question

Model Council — for General Manager · Operations at Contoso Mining Services: what are the three highest-impact strategic moves we should make in the next 12 months in the Indonesian mining services & epc sector, given the peer benchmark above? For each move, give: the rationale, the evidence (cited), the main risk, and a 90-day starter action. Show me where the models agree, where they disagree, and which one I should trust on each move.
Why Model Council matters for GMs: for any decision worth more than ~USD 1M, the cost of one model being wrong dwarfs the cost of asking three. Model Council surfaces the disagreement instead of hiding it.

4.1 · Draft a BCG/McKinsey-style strategy deck

Copilot in PowerPoint drafts a deck in the "one message per slide" style consulting firms use — grounded on your memo and KPI workbook, and laid out on your corporate template.

Task 4.1a🎨 PowerPoint Copilot

Draft an 8-slide GM strategy deck

Open the Copilot pane in PowerPoint. Type / to reference your two source files (05 Contoso_Strategic_Memo.docx and 02 Contoso_KPI_Dashboard.xlsx), then paste:

Create an 8-slide FY26 Project Portfolio & EPC Bid Strategy deck for Contoso Mining Services. Audience: General Manager · Operations and the Operating Committee. Ground every slide on /05 Contoso_Strategic_Memo.docx and the KPI tables in /02 Contoso_KPI_Dashboard.xlsx. Use the Petrosea corporate template "PTP-CC-TPL-G-0006 Wide (16-9) - Presentation Template.pptx" that is currently applied to this file — keep its slide master, fonts, brand colours, logo placement and footer. Do not change them. Apply the strict "one message per slide" consulting style (BCG / McKinsey best practice): (a) Every slide title is an ACTION TITLE — a full sentence that IS the conclusion (e.g. "Same-store sales drag is concentrated in mid-tier department stores"), not a topic ("Same-store sales"). (b) The body proves the title with exactly 3 supporting bullets OR one small evidence table — nothing else. (c) Every slide has a source caption in 8pt grey at the bottom referencing the file and tab/page used. (d) No more than 30 words of body text per slide. No decorative imagery. Slide structure: 1. Title slide — Contoso Mining Services · FY26 Project Portfolio & EPC Bid Strategy · date 2. Executive snapshot — single headline + 3-number callout + RAG status 3. "Where we stand today" — KPI scoreboard (Overburden Removed (Mbcm), Equipment Availability %, Strip Ratio (actual vs plan), Fuel Burn (L per bcm)) 4. "What is changing" — top 3 market forces 5. "Our three FY26 H2 priorities" — one priority per bullet, with named owner 6. "Where we invest first" — 90-day moves with cost & expected benefit 7. "Risks & mitigations" — table: risk, owner, status 8. "The ask" — 1-page decision summary for the Operating Committee Tone: direct, evidence-based, no adjectives, no "leverage / synergy / unlock" language. Use [PLACEHOLDER] for any number not in the source files.
Task 4.1b🎨 Self-critique pass

Tighten the deck — McKinsey-style review

Act as a McKinsey engagement manager reviewing this deck against consulting best practice. For each slide: 1. Score the action title 1–5. If it's less than 5, rewrite it so it IS the conclusion (full sentence). 2. Check that the body proves the title — flag any slide where title and body don't match. 3. Check the "so-what": if the slide doesn't change a decision, mark it for deletion. Return a 1-page review note: slide-by-slide table, plus the single slide to cut, plus the single missing slide we should add. Then apply all the edits.
The BCG/McKinsey check: read only the action titles top-to-bottom. If they tell the complete story without opening any body, the deck is consulting-grade.
