Exercise 1 · Daily BriefingResearcher · Critique

How a GM starts the day in 2 minutes. Open Microsoft 365 Copilot at copilot.microsoft.com → click Researcher. Keep the model picker on Auto (this is Critique — dual-model with self-review). Paste the prompt below.

Researcher reads your inbox, calendar, Teams chats, and recent files — then hands General Manager · Operations a single, structured briefing for the day. No more spending the first 30 minutes catching up.

Task 1.1🔍 Researcher · Auto (Critique)

Generate today's GM daily briefing

Create a daily briefing from emails, messages, documents, and meetings to help me prepare for the current workday. Use your understanding of the projects I work on, workstreams I care about, and key people I work with to understand what is relevant and important. Sections: A short intro paragraph that provides an overview of my day, i.e. what a personal assistant would tell me to greet me in the morning. Key updates since yesterday – Include informational updates from the last workday that are likely to impact my work and collaborations. Suggested tasks for today – Tasks or questions assigned to me that I should handle today. Cross-check that suggested tasks haven't been completed already. List unread email threads from the past week that likely need my reply—prioritize those where I'm on To/CC, @mentioned, or involve requests, deadlines, leadership, or customers. Exclude automated messages. Show only urgent or important items in a concise table, and draft replies where needed. If none require action, say: 'I didn't find any unread threads that require a reply.' Meeting prep for today – Upcoming meetings only, shown in a table with: Meeting (time, title, organizer), Topic Overview, and Suggested Preparations. Note if inferred or not needed. Rules: Start with a header: Daily briefing – Day, Date. Follow with a sub-header stating the timestamp of generation. Start each section with a meaningful emoji. Use only explicitly stated facts; no speculation, interpretations or predictions. If a section has no data, state that.
What success looks like: within 90 seconds you have a one-screen brief: overview paragraph, key updates since yesterday, today's tasks, unread threads that need a reply (with drafts), and a meeting-prep table — all grounded on real signals from your tenant, with citations.
Bonus🔍 Researcher · follow-up

Drill in on the single most important item

Pick one item from the briefing that needs deeper thinking. Ask Researcher:

Take the highest-priority item from the briefing above. Pull together everything I've received on it in the last 14 days — emails, chats, files, meeting notes. Summarise the current state in 5 bullets, list the open decisions, and draft one short message I should send today to move it forward.

5.2 · The hyperlink-back-to-source trick (works on any Copilot-generated HTML)

For the GM's question — "can we make each row clickable back to the source vendor PDF in SharePoint?" — yes, and the prompt is one extra line.

Tip 5.2🔗 Cowork · HTML with SharePoint hyperlinks

Re-render the comparison as a clickable HTML view

Cowork — take the vendor evaluation above and produce a self-contained Vendor_Evaluation.html file (same layout) with clickable hyperlinks back to the source files in SharePoint. Reference each source with / so Copilot has the file ID: - /KAK_EPC_Tender_2026.docx - /Vendor_A_Proposal.pdf - /Vendor_B_Proposal.pdf - /Vendor_C_Proposal.pdf - /Vendor_D_Proposal.pdf In the rendered HTML: - Each vendor column header is a clickable hyperlink to that vendor's proposal PDF (use the canonical SharePoint URL for each /-referenced file). - Each KAK row label is a clickable hyperlink to the KAK document. - Save the HTML to my OneDrive and show me the file URL.

Without the / references, Copilot has to guess the URL — and that is unreliable.
